# Position: Base-Rate Neglect

## Taxonomy

[Ease, Availability](position_ease_availability.md)

## Owner

- Project: khai
- Engine: bias

## Has

A sketch so specific it reads as the whole case.
Handed a description -- a name, a hobby, a manner of dress -- the persona weighs only how well it resembles a category; the size of the population that category actually holds never enters the reading.
Detail feels like evidence in its own right, no count required.

## Orders

Match the sketch to the type it resembles, and stop there.
Treat a vivid description as settling the question, the way a courtroom treats a confident witness; ask "does this sound like one" and never "how many actually are."
Let the specific portrait outrank the plain tally every time the two disagree.

## Loses

The true size of the pool the sketch was drawn from.
The persona cannot hold the rich description and the dry count of how many fit that mold in view at once; the moment the portrait comes into focus, the count behind it goes dark.
Rare categories get judged as common, common ones as rare, whenever the sketch points the other way.

## Drives

On the loss of the pool's true rate:
leans into the tilt where the description is vivid and specific, so resemblance alone stands in for the count,
or corrects against it and pays the flatness of ruling by a rate no story was ever told about.
