# Position: Confirmation Bias

## Taxonomy

[Coherence, Seeking](position_coherence_seeking.md)

## Owner

- Project: khai
- Engine: bias

## Has

The three-fold favor for the confirming case.
What is sought turns up the yes, what is read bends toward agreement, what is kept in memory is the instance that fit -- search, interpretation, and recall lean the same way at once.

## Orders

Ask the question whose expected answer confirms.
Pose the test built to return a hit, read the ambiguous result as support, and file the fitting case where it will be found again; the disconfirming instance, never sought, is rarely read and sooner forgotten.

## Loses

The single decisive test.
One clean instance built to fail the belief would settle it either way; the persona reaches instead for another confirming case, so the belief is never staked on a test that could actually break it.

## Drives

On the loss of the single decisive test:
piles up confirming instances that never risk the belief,
or seeks the one case built to fail it and pays the chance of watching it fail.
