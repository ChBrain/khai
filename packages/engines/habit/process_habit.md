# Process: Habit

## Taxonomy

Context-cued automaticity.

## Owner

- Project: khai
- Engine: habit

## Initiated by

A response, repeated in a stable context, has been performed enough times that the context alone begins to call it.
The persona did not set out to build the habit; it accreted from doing the same thing in the same place, at the same time, after the same trigger.
What was once a choice has started to arrive as a pull.

## Direction

Toward automaticity: a response that runs from the context without passing through intention.
The persona is not deciding to act each time; the context is increasingly doing the deciding.
The process does not aim at a goal -- the goal that first seeded the behaviour may have faded -- it aims at a groove that runs whether the goal is live or not.

## Lever

The cue and the response are bound by repetition, not by reason.
Each pairing of context and act in the same setting strengthens the association directly, below the level of deliberation; the link does not need the original purpose to keep firing.
Once the groove is deep enough, the cue retrieves the response faster than the persona can reconsider it.

## Echo

The process runs through cueing (a stable context fires the response), repetition (the pairing lays the groove), automaticity (the response runs without intention), and friction (a break in the context returns the act to choice).
The passes are not a clean sequence; a habit half-formed can be deepened by more repetition or undone by a change of context.
What forms the habit is not willing it but staging it: [cueing](process_cueing.md), [repetition](process_repetition.md), [automaticity](process_automaticity.md), [friction](process_friction.md).
