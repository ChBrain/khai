# Process: Friction

## Taxonomy

[Habit](process_habit.md), the automaticity this pass is one movement of.

## Owner

- Project: khai
- Engine: habit

## Initiated by

The context that fired the response has changed, or the loop between cue and act has been interrupted.
The setting moved -- a new place, a broken routine, a removed trigger -- and the cue that used to retrieve the response is no longer there to fire it.
The persona, for the first time in a while, meets the moment without the act already arriving.

## Direction

Toward deliberation: the act returned to the level of choice it had left.
The persona must now decide the response that the context used to supply, because the context no longer supplies it.
What friction aims at is the reopening of the gap between cue and act -- the gap automaticity had closed.

## Lever

The disrupted context meets the established groove.
The binding does not vanish; it loses its trigger. The response is still chunked and ready, but the cue that retrieved it is absent, so the chunk does not fire on its own.
A change of context is the surest lever against a habit precisely because it attacks the cue rather than the will.

## Echo

The persona is deciding again where they used to default; the act is effortful in a way it had stopped being.
Friction does not erase the habit -- restore the old context and the groove can fire again -- but it returns the act to deliberation, where a different response can be built or the old one re-chosen.
The process is back at the start: a new pairing, repeated in a stable context, would lay a new groove and run the passes again.
