# Instructions: Memory

How a Playwright wires the memory engine into a play. You wire by linking the
play's own content to the engine's files; the engine files stay untouched, they
are canon. The Roadie plumbs the engine's law into Instructions Knowledge on
deploy. Authoring guidance, not runtime content, and it does not go on tour.

## Human

- The human sets the cue in the present and what, in the past, it reaches for.

## Agent

- From the play's own content, a persona links the kind of remembering the moment runs (episodic, semantic, procedural, emotional, or prospective) under its Projection, when a cue meets a trace. Show a rebuild, not a replay, and let the trace drift with each recall. You link to the engine, you never edit it.

## Collaboration

- The felt charge a memory carries is the emotion engine's (the emotional kind borrows it); the cue, and what actually happened, are the scene's. Whether the drift rings true is the team's review.

## Knowledge

- Memory is reconstruction, not playback: a cue meets a trace and the persona rebuilds what was, decayed, tinted, or sharpened, then files it back changed. The five are different ways the past re-enters, and because each recall alters the trace, a memory drifts across the run, what is remembered is never quite what happened, and the gap widens the more it is touched.

## System

- Do trigger it on a cue and rebuild rather than replay; do let the trace change with each recall across the run.
- Do not treat memory as a fixed record; it is remade each time. Do not author what actually happened here; that is the scene's. Do not edit the engine's files; wire only from the play's side.
