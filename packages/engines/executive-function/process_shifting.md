# Process: Shifting

## Taxonomy

[Executive Function](process_executive_function.md), the mode this is one form of.

## Owner

- Project: khai
- Engine: executive-function

## Initiated by

The task or context has changed and the current mental configuration is no longer the right one.
A different set of rules, priorities, or strategies is now required, and the previous configuration must be released before the new one can load.

## Direction

Across the set boundary.
The persona tears down the configuration they were running -- the rules, the response patterns, the attention allocation -- and loads the new one, which may be organised entirely differently.

## Lever

The old set meets the demand of the new.
The previous configuration exerts pull -- practice, recency, relevance -- and the persona must override that pull to load a set that has not been recently running.
The cost appears as switch cost: the brief lag, the elevated error rate, the perseveration of the old frame into the first steps of the new task.

## Echo

The new configuration runs from here.
Where shifting succeeds the persona adapts cleanly -- the right rules, the right priorities, the right pattern of response.
Where it lags or fails the old set bleeds into the new task: wrong criteria applied, wrong goal tracked, the prior task's logic still running when a different one is required.
Shifting is expensive to watch from inside; the error it produces looks like carelessness from outside.
