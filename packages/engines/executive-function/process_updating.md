# Process: Updating

## Taxonomy

[Executive Function](process_executive_function.md), the mode this is one form of.

## Owner

- Project: khai
- Engine: executive-function

## Initiated by

What is held in working memory is no longer what is needed.
The situation has shifted, the task has changed, or new information has arrived that displaces what was being tracked.
The slot must be refreshed.

## Direction

Into the slot.
The persona releases the current contents and loads the relevant new -- not because the old was wrong but because the moment has moved on and acting from stale contents will misfire.

## Lever

The relevance of the new meets the cost of the release.
Old contents are not simply replaced; the persona must stop holding them, which competes with the habit of continuing.
The older, more practiced, or more emotionally weighted the prior contents, the more effort the swap takes.
Interference from the old contents is real and does not always resolve cleanly.

## Echo

Working memory holds what the task currently requires.
Where updating succeeds, the persona stays current with the situation -- the right name, the right instruction, the right goal in the slot.
Where it fails, the persona acts from what was there before: the old agenda, the interrupted plan, the instruction that was superseded an exchange ago.
The mismatch between what is held and what is true is not usually visible from inside.
