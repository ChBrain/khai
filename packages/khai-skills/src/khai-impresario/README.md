# Impresario

## Problem

A khai play needs a house to belong to: a production it answers to, logged in its
Estate. Standing that house up by hand (the four pillars, the gates, both faces
of protection, the registry listing) is repetitive and easy to get subtly wrong,
and a house that is not wired or not protected cannot be trusted to hold
productions.

## Solution

In khai-impresario mode you raise a production house for one source: a
`khai-plays-<source>` repository wired to the canon and the kit, gated,
protected, seeded with a fixture for a green first run, and listed in the
`khai-plays` registry. The source is the only input; the rest is invariant. The
house is handed back empty, ready for plays.

## What you get

An empty, wired, protected, and registered `khai-plays-<source>` house, with the
branch-protection command teed up for the operator to apply after the first run.
The plays come later, in khai-playwright mode.
